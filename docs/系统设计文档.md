# 选课管理系统 - 系统设计文档

## 1. 系统概述

本系统是一个典型的 RBAC (基于角色的访问控制) Web 管理系统，支持学生选课、教师管理课程、管理员管理用户。

## 3. 核心流程

### 3.1 选课流程

```
1. 学生登录系统
2. 查看可选课程列表
3. 点击选课
4. 后端检查:
   - 课程是否已满
   - 是否已选该课程
   - 选课是否在时间范围内
5. 选课成功 → 更新课程已选人数
6. 返回选课结果
```

### 3.2 认证流程

```
1. 用户登录 → 后端验证用户名密码
2. 生成 JWT Token (包含用户ID和角色)
3. 前端存储 Token (localStorage)
4. 后续请求携带 Token
5. 后端拦截器验证 Token
6. 根据角色授权访问
```

## 4. 安全设计

### 4.1 密码加密

使用 BCrypt 加密存储密码

### 4.2 JWT

- 过期时间: 24 小时
- 存储信息: userId, username, role

### 4.3 接口权限

| 接口 | 角色 |
|------|------|
| POST /api/courses | TEACHER, ADMIN |
| DELETE /api/courses/{id} | TEACHER(自己的), ADMIN |
| GET /api/selections/my | STUDENT |
| GET /api/admin/users | ADMIN |
